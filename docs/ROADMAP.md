# Scout 产品路线图

## Phase 1: 多样化需求载体
让 Scout 能理解各种格式的需求文档。

### 1.1 URL 内容抓取
- 输入飞书/Notion/在线文档链接，自动抓取网页内容
- 支持表格类页面的结构化解析
- 状态：✅ 已完成

### 1.2 文件上传
- 支持 PDF、DOCX、图片（截图/原型图）上传
- PDF 文本提取 + 图片提取
- DOCX 内容解析
- 图片通过多模态 LLM 理解
- 状态：✅ 已完成

### 1.3 表格数据支持
- 支持 CSV/Excel 上传
- 表格内容结构化解析为需求描述
- 状态：✅ 已完成

---

## Phase 2: 大脑图文认知系统
让 Scout 的"大脑"具备图像储存、消费和组合能力。

### 2.1 图像存储
- 碎片模型扩展：支持图片附件（本地存储 + 路径引用）
- 图片缩略图生成
- 状态：✅ 已完成

### 2.2 图像理解（消费）
- 多模态 LLM 调用能力（GPT-4V / Claude Vision）
- 图片内容提取和描述生成
- 图片与文字碎片的关联
- 状态：✅ 已完成

### 2.3 图像联想（组合）
- "想到一个概念 → 联想到具象照片"的检索能力
- 图文混合的知识图谱展示
- 状态：✅ 已完成

---

## Phase 3: GEP 执行协议
从"步骤复读机"到"有经验的测试搭档"。

### 3.1 GEP 核心
- Gene（基因）：测试意图 + 验收条件
- Capsule（胶囊）：执行路径 + 环境指纹
- Insights（经验）：执行发现沉淀
- 状态：✅ 已完成

### 3.2 双通道执行
- Accessibility Tree：结构化理解，精准定位
- 截图验证：视觉确认，像人一样看结果
- 状态：✅ 已完成

### 3.3 环境自适应
- 高匹配 → 复用策略
- 中匹配 → 适配策略
- 低匹配 → 探索策略
- 状态：✅ 已完成

### 3.4 可视化测试报告
- 关键步骤自动截图
- 图文并茂的测试报告展示
- 通过/失败一目了然
- 状态：✅ 已完成

### 3.5 浏览器调通
- 修复现有 Playwright 执行器的问题
- 稳定的 Chrome 接管和登录态继承
- 状态：✅ 已完成

Phase 1.1 ✅ → Phase 3.1-3.5 ✅ → Phase 1.2 → Phase 2.1 → Phase 2.2 → Phase 2.3 → Phase 1.3

## 已完成总结

### Phase 1.1: URL 内容抓取
- 支持飞书、Notion、语雀、通用网页抓取
- 自动检测 URL 输入并抓取内容
- 抓取后自动提取正文和标题

### Phase 3: GEP 执行协议
- **Gene 模型**：从测试用例提取测试意图和验收条件
- **Capsule 模型**：记录成功执行路径和环境指纹
- **Insights 模型**：沉淀执行中的发现和经验
- **双通道方案**：Accessibility Tree + 截图验证
- **三种执行策略**：复用、适配、探索
- **测试报告**：关键步骤截图，可视化报告
