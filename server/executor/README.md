# server/executor/

用例执行器，用 Playwright 接管用户 Chrome 进行自动化测试。

## 文件

| 文件 | 地位 | 功能 |
|------|------|------|
| pi-engine-runner.js | 智能执行器 | 脚本优先回放、旧动作别名兼容、意图理解 → 知识检索 → 跨端路径规划 → Web/手机自主执行 |
| enhanced-runner.js | 快速执行器 | 直接解析步骤执行，速度快 |
| batch-executor.js | 批量编排 | 根据配置选择执行器，生成带 Prism Engine 日志和原始用例详情的报告 |
| auto-runner.js | 自然语言执行器 | 接收自然语言指令执行 |
| playwright-runner.js | 底层工具 | Playwright 基础操作封装 |

## 智能执行流程 (PI Engine)

```
测试用例
    ↓
┌─────────────────────────────────────────────────┐
│  阶段 1: 意图分析                                 │
│  - 理解用例要验证什么功能/场景                      │
│  - 识别测试目标、关键操作、风险等级                   │
└─────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────┐
│  阶段 2: 知识检索                                 │
│  - 调用"大脑"检索相关记忆碎片                       │
│  - 提取 URL、账号信息、操作提示                      │
└─────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────┐
│  阶段 3: 路径规划                                 │
│  - 结合意图和知识，规划执行路径                       │
│  - 生成具体、可执行的步骤列表                        │
└─────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────┐
│  阶段 4: 自主执行                                 │
│  - PI Agent 按规划路径自主执行                      │
│  - 每一步调用浏览器工具完成                          │
│  - 遇到问题自动调整策略                             │
└─────────────────────────────────────────────────┘
    ↓
执行结果
```

## 与快速模式的区别

| 维度 | 智能模式 (PI Engine) | 快速模式 (Enhanced) |
|------|---------------------|-------------------|
| 理解能力 | 理解测试意图 | 按字面执行 |
| 知识利用 | 调用大脑获取知识 | 无 |
| 路径规划 | 智能规划执行路径 | 固定步骤 |
| 执行方式 | PI Agent 自主执行 | 硬编码操作 |
| 速度 | 较慢（需思考） | 快 |
| 适用场景 | 复杂业务、需要理解 | 简单重复操作 |

## 跨端用例

步骤使用 `[Web]`、`[后台]`、`[手机]`、`[移动端]`、`[小程序]` 或 `[H5]` 标记执行端。Web 端由 Playwright 操作 Chrome，手机端通过 USB 或无线 ADB 操作 Android 真机；快速模式遇到混合端步骤时会自动升级为智能模式。
